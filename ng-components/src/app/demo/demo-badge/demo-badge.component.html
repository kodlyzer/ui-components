Badge Component

Test Inputs (JSON DATA)

  • Description

    • A badge can serve as a central element for the identity of a website and emerge as a reminder feature on every sub-site. In contrast to the button(which is a useful element for call to action), badge is usually more than a call to action. Generally, the badge is used for its importance as a home button, but there are more possibilities for use.

  • Configuration Options

    • Use this Selector in your component to include Badge Component.

                <kdi-badge [badge]="demoBadge">
            </kdi-badge>
  • How to use ?

    • Follow the following format to use it.

                 demoBadge= {  color:"name of the color (from the list of colors provided by KDI)",  
              label:"name of the label you want to display "   } ;                         
           Here we have used :
             demoBadge1= {  color: 'primary',  label:'Primary' } ;
             demoBadge2= { color:'success', label:'Success'  } ;
             and so on.    
             
             Note: If you don't provide the "color" value, by default it will give the "foam" as color.

    The list of colors present now are primary, secondary, success, danger, foam, navy and warning.
